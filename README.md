# IMSI Catcher Detector - Android Application\n\n## Overview\n\nIMSI Catcher Detector is a comprehensive Android application designed to identify and alert users to the presence of fake cellular base stations (IMSI catchers). The app combines advanced signal analysis, cell tower monitoring, encryption verification, and network behavior profiling to provide real-time threat detection with a modern Material You interface.\n\n## Features\n\n### Core Detection Capabilities\n\n**Encryption Monitoring**\n- Detects A5/0 (no encryption) - the most critical IMSI catcher indicator\n- Identifies weak ciphers (A5/1, A5/2, 128-EEA1)\n- Detects cipher downgrade attacks\n- Monitors encryption status changes\n\n**Cell Tower Analysis**\n- LAC/TAC consistency verification\n- CID validity checking\n- ARFCN/EARFCN/UARFCN validation for all network types\n- Tower hopping detection (rapid cell changes)\n- Neighbor cell relationship analysis\n- Impossible LAC/TAC combination detection\n\n**Signal Strength Analysis**\n- Sudden signal change detection (>15 dBm threshold)\n- Oscillating signal pattern recognition\n- Unusually strong signal detection\n- Signal quality analysis (RSRP/RSRQ mismatch)\n- Timing advance anomaly detection\n- Statistical trend analysis\n\n**Network Behavior Profiling**\n- LTE/3G/2G downgrade detection\n- Manual network selection monitoring\n- Roaming status tracking\n- Network type consistency verification\n\n**Protocol Analysis**\n- Silent SMS (Class 0) detection\n- WAP Push anomaly detection\n- Suspicious message pattern recognition\n- Call pattern monitoring\n\n### User Interface\n\n**Real-time Dashboard**\n- Color-coded threat levels (Green/Yellow/Orange/Red)\n- Current cell tower information display\n- Signal strength visualization\n- Encryption status indicator\n- Active threat alerts\n\n**Threat History**\n- Chronological list of detected anomalies\n- Detailed threat information with timestamps\n- Severity scoring and recommendations\n- Historical trend analysis\n\n**Settings**\n- Background monitoring toggle\n- Notification preferences\n- Auto-baseline reset option\n- Data management controls\n- App information\n\n**Interactive Map** (optional)\n- Cell tower location visualization\n- Neighbor cell display\n- Signal strength coverage heatmap\n- Historical tower tracking\n\n## Technical Architecture\n\n### Detection Engine\n\nThe app implements a unified threat scoring system that combines multiple independent detection modules:\n\n```\nThreat Score = (Encryption × 0.45) + (Cell Consistency × 0.30) + \n               (Signal Anomaly × 0.15) + (Protocol Anomaly × 0.10)\n```\n\n**Threat Levels:**\n- Green (0-20): Normal operation\n- Yellow (21-50): Potential anomaly\n- Orange (51-75): Significant threat\n- Red (76-100): Critical threat\n\n### Technology Stack\n\n**Language & Framework**\n- Kotlin with coroutines for asynchronous operations\n- Jetpack Compose for modern reactive UI\n- Material Design 3 with dynamic theming\n\n**Data Storage**\n- Realm database for efficient local persistence\n- Encrypted storage of sensitive data\n- Automatic data cleanup policies\n\n**Background Services**\n- Foreground service for continuous monitoring\n- TelephonyCallback (Android 12+) for efficient state updates\n- WorkManager for periodic tasks\n\n**Android APIs**\n- TelephonyManager for cell information\n- ServiceState for network registration\n- CellInfo classes for detailed cell data\n- Location API for triangulation\n- Google Maps API for visualization\n\n## Permissions\n\nThe app requires the following permissions:\n\n- `READ_PHONE_STATE` - Access phone state\n- `READ_PRECISE_PHONE_STATE` - Android 12+ detailed cell info\n- `ACCESS_FINE_LOCATION` - Cell tower triangulation\n- `ACCESS_COARSE_LOCATION` - Approximate location\n- `RECEIVE_SMS` - Monitor incoming SMS\n- `READ_SMS` - Access SMS database\n- `INTERNET` - Network communication\n- `POST_NOTIFICATIONS` - Alert notifications\n- `FOREGROUND_SERVICE` - Background monitoring\n\n## Installation\n\n### Building from Source\n\n```bash\n# Clone the repository\ngit clone https://github.com/yourusername/imsi-catcher-detector.git\ncd imsi-catcher-detector\n\n# Build release APK\n./gradlew assembleRelease\n\n# Build debug APK\n./gradlew assembleDebug\n```\n\n### Requirements\n\n- Android Studio Flamingo or later\n- Android SDK 30+ (API level 30)\n- Kotlin 1.9.10+\n- Gradle 8.2.0+\n\n## Usage\n\n### Starting the App\n\n1. Launch the IMSI Detector app\n2. Grant required permissions when prompted\n3. The app will begin monitoring cellular network immediately\n4. Threat level indicator will display current status\n\n### Monitoring\n\n- **Dashboard**: View current threat level and cell information\n- **History**: Review past threats and anomalies\n- **Settings**: Configure monitoring preferences\n- **Notifications**: Receive real-time alerts for detected threats\n\n### Interpreting Results\n\n**Green (Safe)**\n- No threats detected\n- Normal network operation\n- Continue normal usage\n\n**Yellow (Caution)**\n- Minor anomalies detected\n- Continue monitoring\n- Verify network legitimacy\n\n**Orange (Warning)**\n- Significant threat indicators present\n- Verify network with provider\n- Avoid sensitive transactions\n\n**Red (Critical)**\n- High probability IMSI catcher detected\n- Move to safe location immediately\n- Contact network provider\n- Consider using VPN\n\n## Database Schema\n\n### CellTowerRecord\n- Timestamp, LAC, TAC, CID\n- Signal metrics (RSSI, RSRP, RSRQ)\n- Encryption status and cipher algorithm\n- Network type and operator information\n- Location coordinates\n- Neighbor cell list\n\n### ThreatEvent\n- Threat type and severity score\n- Threat level classification\n- Affected parameters and values\n- Description and recommendations\n- Associated cell tower record\n\n### SMSLog\n- Sender, content, timestamp\n- Message classification\n- Message class (for silent SMS detection)\n- Port information (for binary SMS)\n\n### BaselineProfile\n- Location-based or time-based baseline\n- Expected cell tower information\n- Signal strength ranges\n- Typical neighbor cells\n- Trust level rating\n\n## Performance Optimization\n\n- **Battery Efficient**: Uses callbacks instead of polling\n- **Memory Optimized**: Streams data instead of loading entire histories\n- **Network Efficient**: Caches map tiles and cell tower data\n- **Database Optimized**: Automatic cleanup of old records\n\n## Security Considerations\n\n- All data stored locally on device\n- No external data transmission without consent\n- Realm database encryption support\n- Sensitive parameters never logged to system logs\n- Secure permission handling\n\n## Troubleshooting\n\n### No Cell Information Displayed\n- Verify permissions are granted\n- Check device has active cellular connection\n- Ensure device is not in airplane mode\n- Restart the app\n\n### Frequent False Positives\n- Verify you're in a legitimate network area\n- Check signal strength is reasonable\n- Review threat details for specific anomalies\n- Consider resetting baseline for new location\n\n### Battery Drain\n- Reduce background monitoring frequency in settings\n- Disable location-based features if not needed\n- Check for excessive logging in debug mode\n\n## Future Enhancements\n\n- Machine learning-based anomaly
